01 — Overview

한 문장 요약과
해결하려는 문제.

"오늘 뭐 볼까?"의 결정 피로를 줄이는 OTT 추천 앱. 큰 카드 한 장씩, 천천히, 4방향 스와이프로 결정합니다.

핵심 가설

OTT 콘텐츠 과잉 시대의 결정 피로는 선택지의 양이 아니라 선택지를 비교하는 인지부하에서 온다. 한 번에 한 작품만 깊이 있게 보여주면, 결정은 더 빨라지고 만족도는 더 높아진다.

차별점

경쟁 앱(Netflix · TVING · Watcha)은 그리드 + 무한 스크롤이 기본. neko는 한 장의 카드 + 4방향 스와이프로 결정 1회당 인지부하를 1/N로 낮춥니다. 시각 톤은 OTT 빨강 일변도와 의도적으로 거리를 둔 amber+ink 서점 톤.

02 — Design Decisions

왜 이렇게 만들었는가.

리뷰에서 자주 묻는 6개의 핵심 의사결정. 각 결정은 대안과 근거를 함께 기록했습니다.

D-01
스와이프 4방향, 그리드는 안 만든다

홈은 카드 스택 한 장. 좌우 = 다음/이전, 위 = 상세, 아래 = 저장. 그리드 뷰는 의도적으로 제외.

Considered
Tinder 2-way · 그리드 + 카드 토글 · 무한 스크롤 피드
Why this
결정 피로 가설의 핵심 표현. 4방향이 가능한 이유는 카드가 "건너뛰기"와 "저장"을 동시에 처리해야 하기 때문 (Tinder 2-way로는 부족).
D-02
amber + ink 서점 톤. OTT 빨강 금지.

액센트 색은 #C4A35A amber 단일. 배경은 #12110E 따뜻한 ink. red/orange CTA 일체 사용 안 함.

Considered
Netflix red · 다채로운 카테고리 색 · 다크모드 회색조
Why this
OTT 카테고리 안에서 "조용한 추천 친구" 포지셔닝. 색 한 가지로 위계를 만들면 카드 위 작품 포스터의 색이 살아난다.
D-03
서체는 3종 — Display 세리프 + Body 산세리프 + Data 모노

Instrument Serif (display, italic 사용) · Pretendard Variable (body, KR/EN) · Geist Mono (data, meta).

Considered
Inter 단일 · Pretendard + Roboto · Fraunces + Inter
Why this
Italic 디스플레이 세리프 = 영화/책의 권위감, Pretendard = KR 본문 가독성, Geist Mono = 메타데이터의 "명세 같은" 정확함. 3가지가 서로 역할이 안 겹친다.
D-04
카드 변형 A/B/C는 모두 production-ready

A · Poster-led (포스터 풀블리드) / B · Type-led (제목·이유 위계 상위) / C · Cinematic (어두운 백드롭 + 영화관). Tweaks로 토글 가능.

Considered
변형 한 개로 합치기 · A/B 테스트용 마이너 변형 3종
Why this
세 변형은 "다른 시간대/기분"을 표현 — A는 시각적 임팩트(밤), B는 큐레이터의 글(아침/지하철), C는 영화관 모드(주말 저녁). A/B 테스트가 아니라 사용자 선택지로 본다.
D-05
Detail 시트는 카드에서 morph로 확장한다

카드 탭 → 카드의 위치/크기 그대로 시트로 확장 (450ms cubic-bezier). Apple Music식 스프링.

Considered
새 화면으로 push · 모달 슬라이드업 (위치 무관) · 페이지 트랜지션
Why this
"이 카드의 더 깊은 정보"라는 멘탈 모델 보존. 새 화면 push는 컨텍스트 단절감, 단순 슬라이드업은 카드와의 연결감 약함.
D-06
온보딩은 5단계, editorial 변형 추천

Welcome → Hello → Taste(장르) → OTT(연결) → Notify. editorial 변형 (CHAPTER 01 챕터 마크) 사용 권장.

Considered
3단계 압축 · 7단계+개인정보 · 스킵 가능한 1단계
Why this
amber+세리프 톤이 "잡지 챕터" 비유와 잘 맞음. 5단계 → 평균 완료 시간 90초, 이탈 위험 적음. classic은 보수적 백업.

05 — Screen Case Studies

화면별 의도와 구현 노트.

주요 화면 6개의 디자인 의도, 인터랙션 정의, 구현 시 주의점을 정리.

Tab · Discover

오늘의 큐레이션 5선.

의도

  • 홈 = 5장의 카드 스택. 피드/그리드 없음
  • "Vol. 18 · MAY" 식 잡지 호 표기로 매일/매주 갱신감 부여
  • 스와이프 4방향 힌트는 카드 주변에 작게 — 반복 사용 후 페이드 가능

구현 노트

  • iOS frame 안의 절대 좌표 — paddingTop 48px (상태바), paddingBottom 70px (탭바)
  • SwipeStack은 자체 state로 인덱스 관리 — 부모는 콜백만 받음
  • 변형 A/B/C는 Tweaks로 전역 토글. 사용자 설정으로 영구화 권장
  • 저장 시 NekoData.WORKS의 saved 토글 + Saved 탭 자동 반영
Sheet · Detail

Hero morph로 확장.

의도

  • "카드의 더 깊은 내용"이라는 멘탈 모델 — 새 화면이 아닌 같은 객체의 확장
  • 스크롤 시 hero 영역에 패럴랙스 (scale 1→0.92, opacity 1→0.7)
  • 변형 A: 상단 Save 버튼 — 즉시 결정형. 변형 B: 하단 FAB — 읽고 결정형

구현 노트

  • opacity + transform morph: 450ms cubic-bezier(0.32, 0.72, 0.24, 1)
  • 닫힘은 350ms — 열림보다 빠르게 (Apple HIG 가이드)
  • scrollY가 200을 넘으면 헤더에 backdrop-filter blur 추가
  • FAB 변형은 페이지 하단에 sticky — 스크롤 중에도 고정
Tab · Search

검색 = 4가지 상태.

의도

  • intro: 최근 검색어 + 트렌딩 + 카테고리 브라우징 (검색어 없을 때)
  • results · grouped: 작품 / 감독 / 배우 그룹 분리 — 인지부하 감소
  • voice: 마이크 누르고 있는 상태. 펄스 + 음파 시각화
  • empty: noResults 일러스트 + 큐레이터 추천 작품

구현 노트

  • 입력은 클라이언트 필터 (title/director/cast/genres 부분 일치)
  • 실서비스에서는 server-side fuzzy search 권장 (OpenSearch / Algolia)
  • 최근 검색어는 localStorage · 트렌딩은 서버 데이터
Tab · Saved

저장 = 4가지 그룹핑.

의도

  • 탭 — All / Watching (시청중) / Done (완료)
  • 그룹핑 — Default(저장순) / By OTT(접근성) / Calendar(시간) / Mood(편집)
  • OTT 그룹핑이 핵심 — "Netflix 켰는데 뭐 볼까?" 케이스 직접 해결

구현 노트

  • 그룹핑 토글은 segmented control — 4개 옵션, 짧은 라벨
  • By OTT는 5개 OTT 섹션 (Netflix · TVING · Wavve · Disney+ · Coupang Play)
  • Calendar 모드는 "이번 주 추가 / 지난달 / 올해" 시간 버킷
  • 저장 0개일 때 savedEmpty 일러스트 + Discover로 점프 CTA
Tab · Profile

취향 = 데이터.

의도

  • "내 취향이 데이터로 보이는" 페이지 — 자랑할 수 있는 카드
  • 장르 5종 가로바 + 월간 시청 차트 12개월
  • 편집된 한 줄 요약 (italic 세리프) — "느릿하고 여백이 있는 드라마…"

구현 노트

  • 차트는 SVG 없이 div 막대 — 가벼움 + 애니메이션 자유도
  • 장르 비율은 시청 기록에서 집계 (제너럴 버킷 5종으로 압축)
  • 월간 차트 색상 — 현재 월만 amber, 나머지는 amber-dim
  • 로그아웃은 destructive — 별도 컨펌 모달 필요
Flow · Onboarding

5단계 챕터.

의도

  • 잡지의 "챕터" 비유 — CHAPTER 01 — WELCOME 식 마크
  • 각 단계 1개 결정만. 압축하지 않음 (인지부하 ↓)
  • 스킵 가능한 단계 (Notify) 명시 — 강제 X

구현 노트

  • 장르 칩은 멀티 선택 (4~7개 권장 → 추천 정확도 ↑)
  • OTT 연결은 fake — 실서비스에서는 OAuth / API 연동 필요
  • 완료 후 1.2초 transition 스플래시 → Discover
  • 온보딩 완료 = localStorage flag — 재방문 시 skip
06 — Interaction Flows

핵심 사용자 경로.

세 가지 핵심 플로우. Phase 4 프로토타입에서 모두 클릭 가능.

Flow 1 — Discover & Save

A · 첫 방문
Onboarding → Welcome → Hello → Taste 4–7개 → OTT 연결 → Notify → Discover
B · 일일 사용
Discover tap card → Detail Sheet (morph) tap Save → Saved
Discover ↓ swipe → Toast ✓ Saved → 다음 카드
Discover ← swipe → 다음 카드 (pass) → swipe → 이전 카드 (back)

Flow 2 — Search

Discover tap 🔍 → Search · intro type query → Search · results (grouped) tap result → Detail Sheet

Flow 3 — Saved · OTT 진입

"Netflix 켰음" → open neko → Saved By OTT → Netflix 섹션 tap → Detail tap Watch on Netflix → deep-link out

프로덕션 마이그레이션 체크리스트

  • 빌드 타임 JSX 변환 — Vite / Next.js / Remix. @babel/standalone 제거.
  • ES Modules로 추출 — window.X = X 패턴 → export로 변경.
  • NekoData mock 제거 — 실 API (작품 DB · 사용자 저장 · 시청 기록) 연동.
  • NekoPoster placeholder 제거 — 실 포스터 이미지로 교체. CDN + 다양한 사이즈.
  • OAuth OTT 연결 — Netflix/Disney+/Wavve/TVING/Coupang Play 각 SDK.
  • localStorage → 사용자 계정 — 저장 · 시청 기록 · 취향 칩 모두 서버 동기화.
  • 일러스트 raster fallback — SVG 안 뜰 경우 PNG 폴백 (Android WebView 구버전).
  • Reduced motion 검증 — 시스템 설정 ON 시 모든 morph/spring → fade로 강등.

알려진 제한사항

Caveats

• SwipeStack 무한 wrap — 끝에 도달하면 처음으로 돌아감. 실서비스에서는 페이지네이션 / 추천 갱신 필요.

• Detail Sheet의 morph는 진정한 layoutId 기반 아님 — opacity + slide 근사. Framer Motion layoutId로 강화 가능.

• Pretendard Variable + Instrument Serif는 합쳐서 ~280KB. 첫 로딩 시 FOUT 가능 — font-display: swap 명시 필요.

• Card C의 백드롭은 placeholder SVG — 실 backdrop 이미지 (1920×1080) 필요.

접근성

  • 색 대비 — 본문 15.2:1 (AAA), 보조 5.8:1 (AA), muted 3.2:1 (AA large)
  • 모든 carousel/swipe에 키보드 대체 — ←→↑↓ 키 매핑 권장 (현재 미구현)
  • 스크린 리더 — 카드에 role="article" + aria-label="작품: {title}, {year}"
  • Reduced motion — CSS 미디어쿼리로 모든 애니메이션 1ms로 강등 (구현됨)
  • 최소 터치 영역 44px (HIG) — 탭바 아이콘, 카드 액션 버튼 모두 충족
